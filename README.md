# การบ้าน: Asyncio, Threading และ Process Pool ใน Python

โปรเจกต์นี้เป็นการทดลองเขียนโปรแกรมขนาดเล็กเพื่อทำความเข้าใจการทำงานของ Concurrency และ Parallelism ในภาษา Python ผ่านการจำลองระบบเบื้องหลังของการพัฒนาเกม (Game Processing Simulator)

## เทคโนโลยีที่เลือกใช้และวัตถุประสงค์

### 1. Asyncio (Asynchronous I/O)
* **จำลองการทำงาน:** การเช็คสถานะและ Latency ของเซิร์ฟเวอร์เกม
* **ทำไมถึงใช้:** `asyncio` เหมาะกับงานประเภท Network I/O ที่โปรแกรมต้องรอการตอบกลับจากภายนอก การใช้ `async/await` ทำให้โปรแกรมสามารถสลับไปทำงานอื่นได้โดยไม่ต้องรอให้เซิร์ฟเวอร์ตอบกลับจนเสร็จ (Non-blocking)

### 2. Threading
* **จำลองการทำงาน:** การโหลดไฟล์ Assets (เช่น ไฟล์แอนิเมชันการรีโหลดกระสุน) จากฮาร์ดดิสก์เข้าสู่หน่วยความจำ
* **ทำไมถึงใช้:** `threading` เหมาะกับงานประเภท Disk I/O เนื่องจากในระหว่างที่ OS กำลังอ่านไฟล์จากดิสก์ (ซึ่งช้ากว่า CPU มาก) Thread อื่นๆ สามารถทำงานขนานกันไปได้ ทำให้โหลดไฟล์หลายๆ ไฟล์เสร็จพร้อมกันในเวลาที่ลดลง

### 3. Process Pool (Multiprocessing)
* **จำลองการทำงาน:** การคำนวณฟิสิกส์วิถีกระสุน (Bullet Physics Trajectory)
* **ทำไมถึงใช้:** การคำนวณทางคณิตศาสตร์หนักๆ เป็นงานประเภท CPU Bound หากใช้ Threading ใน Python จะติดข้อจำกัดของ GIL (Global Interpreter Lock) จึงต้องเลี่ยงมาใช้ `multiprocessing.Pool` เพื่อกระจายการคำนวณไปยัง CPU Core อื่นๆ อย่างแท้จริง (True Parallelism)
